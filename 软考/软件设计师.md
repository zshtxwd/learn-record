### 1.3码制

- 原码：原码也能进行运算，但是遇到负数的情况会很麻烦，原码进行带有负数的运算时，要比较大小，判断符号，选择加法还是减法，确定结果符号
- 反码：反码进行运算时，最高位进位需要再加回来，每次都要额外处理最高位进位，硬件不够简单
- 补码：最高位进位直接舍弃最高位，补码可以让运算更方便，现代计算机保存的就是补码，这样计算器内部只需要一个加法器就能进行加法，减法和符号运算。补码的优势就在于他统一的运算
- 移码

正数

```
原码=反码=补码
移码为符号位取反
```

负数

```
反码为原码符号位不变，其余位取反
补码为反码+1，符号位参与运算
移码为补码的符号位取反

tip：负数的补码转原码，可以符号位不变，数值位从右往左数第1个1的前1位开始全部取反

例如：
补码 11111100
符号位的1不动
从右往左的第一个1和他的右边部分不动，即100不动
其他位取反，然后拼接1 0000 100
原码就是 10000100
```

### 1.6校验码

---

#### 一、奇偶校验（Parity Check）

##### 1. 原理
通过添加1位校验位，使1的个数满足：
- 奇校验：1的个数为奇数
- 偶校验：1的个数为偶数

##### 2. 示例

信息位：
```
1001
```

奇校验：
```
11001
```

偶校验：
```
01001
```

##### 3. 特点
- 可检测奇数位错误
- 不可纠错
- 简单低成本

---

#### 二、CRC循环冗余校验

##### 1. 核心思想
将数据看作多项式，用生成多项式 G(x) 做模2除法（异或运算）。

---

##### 2. 生成多项式示例
```
G(x) = x^3 + x + 1
```

对应二进制：
```
1011
```

---

##### 3. 编码示例

###### 数据：
```
1101
```

###### 生成多项式：
```
1011
```

###### 步骤：
1. 补3个0（阶数=3）
```
1101000
```

2. 模2除法（略计算过程）

3. 得到余数：
```
余数 = 011
```

4. 最终发送：
```
1101 + 011 = 1101011
```

---

##### 4. 检验过程
接收端用同一个 G(x) 去除：

- 余数 = 0 → 正确
- 余数 ≠ 0 → 出错

---

##### 5. 特点
- 检错能力强
- 不可纠错
- 广泛用于网络协议

---

#### 三、海明码（Hamming Code）

##### 1. 能力
- 可检错
- 可纠错（单比特错误）

---

##### 2. 校验位计算
满足：
```
2^r ≥ m + r + 1
```

---

##### 3. 示例（4位数据）

###### 信息位：
```
1011
```

###### 插入校验位位置：
```
位置：1 2 3 4 5 6 7
内容：R1 R2 I1 R4 I2 I3 I4
```

实际排列：
```
? ? 1 ? 0 1 1
```

---

##### 4. 校验计算（示意）

- R1负责：1,3,5,7
- R2负责：2,3,6,7
- R4负责：4,5,6,7

计算后得到：
```
R1=0, R2=1, R4=1
```

最终码：
```
0110011
```

---

##### 5. 纠错示例

假设第5位出错：

接收：
```
0110001
```

重新校验得到：
```
S = 101₂ = 5
```

说明：
```
第5位错误
```

翻转即可纠正。

---

##### 6. 特点
- 可纠正1位错误
- 不可靠纠正多位错误
- 利用二进制位置编码定位错误

---

#### 四、总结对比

| 方法 | 检错 | 纠错 | 特点 |
|------|------|------|------|
| 奇偶校验 | ✔ | ✘ | 简单 |
| CRC | ✔✔✔ | ✘ | 强检错 |
| 海明码 | ✔✔ | ✔（1位） | 可定位错误 |
